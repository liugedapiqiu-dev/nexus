## 🔄 数据流向

### 典型请求流程

```
用户提问
    │
    ▼
OpenClaw Gateway (接收消息)
    │
    ▼
LLM (Qwen3.5-Plus) 分析意图
    │
    ├─→ 需要检索记忆？
    │       │
    │       ▼
    │   vectorbrain-memory-search
    │       │
    │       ▼
    │   VectorBrain 知识记忆检索
    │       │
    │       ▼
    │   返回相关记忆 → LLM 整合答案
    │
    ├─→ 需要执行任务？
    │       │
    │       ▼
    │   对应技能 (如 office-automation)
    │       │
    │       ▼
    │   执行脚本 → 返回结果
    │
    └─→ 普通对话？
            │
            ▼
        LLM 直接回答
```

### 路径边界补充

- `~/.openclaw/workspace/memory/` 是**文件记忆层**，不是废弃目录。
- `~/.vectorbrain/` 是**长期数据库记忆层**。
- `~/.openclaw/memory/` 下的旧路径只可视为历史遗留，不能再当当前主记忆路径。
- 统一规范见：`~/.openclaw/workspace/PATH_SPEC.md`

## 🔧 核心组件说明

### 1. OpenClaw Gateway
**作用:** 消息网关，负责接收和路由消息

**配置:**
- 主配置：`~/.openclaw/openclaw.json`
- 旧/兼容配置：`~/.openclaw/config.json`
- 环境变量：`~/.openclaw/.env`

**启动命令:**
```bash
openclaw gateway start
openclaw gateway restart
openclaw gateway status
```

### 2. VectorBrain 大脑
**作用:** 记忆存储和检索系统

**核心数据库:**
| 数据库 | 用途 | 记录数 |
|--------|------|--------|
| episodic_memory.db | 情景记忆（对话历史） | 10,018+ |
| knowledge_memory.db | 知识记忆（提炼知识） | 2,137+ |
| reflections.db | 反思记录（经验教训） | 8,031+ |
| task_queue.db | 任务队列 | 29 |
| goals.db | 目标系统 | 1 |

**检索命令:**
```bash
python3 ~/.vectorbrain/connector/vector_search.py "{查询内容}"
```

### 3. 技能系统
**作用:** 扩展能力，实现特定功能

**技能类型:**
- **核心技能:** 记忆检索、健康监控、记忆提炼、任务反思
- **自动化工具:** 归档系统、知识去重、技能检查
- **业务技能:** Office 自动化、财经分析、桌面控制

**技能配置:**
- `skill.json` - 技能定义和触发器
- `SKILL.md` - 技能使用说明
