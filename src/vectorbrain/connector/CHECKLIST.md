# 📋 配置检查清单 (CHECKLIST.md)

**位置：** `~/.vectorbrain/connector/CHECKLIST.md`  
**用途：** 确保所有配置和任务执行前 100% 完整准确  
**原则：** 不检查，不执行；不验证，不交付

---

## 🎯 适用场景

1. **创建新技能** (skill.json)
2. **修改核心配置** (openclaw.json, config.json)
3. **执行重要任务** (任务队列中的任务)
4. **部署新脚本** (VectorBrain connector 脚本)
5. **系统升级后** (OpenClaw/VectorBrain 更新)

---

## ✅ 三轮自检制度

### 第一轮：完整性检查 (创建时)

**技能配置检查：**
- [ ] 第一优先级技能 = `vectorbrain-memory-search`
- [ ] skill.json 包含完整的 `triggers` 配置
- [ ] skill.json 包含完整的 `actions` 配置
- [ ] 文件路径使用 `~/.vectorbrain/` 前缀（不用旧路径）
- [ ] 已设置正确的文件权限 (chmod +x)

**记忆系统路径检查：**
- [ ] `episodic_memory_db` = `~/.vectorbrain/memory/episodic_memory.db`
- [ ] `knowledge_memory_db` = `~/.vectorbrain/memory/knowledge_memory.db`
- [ ] `reflections_db` = `~/.vectorbrain/reflection/reflections.db`
- [ ] `tasks_db` = `~/.vectorbrain/tasks/task_queue.db`
- [ ] `goals_db` = `~/.vectorbrain/goals/goals.db`
- [ ] `opportunities_db` = `~/.vectorbrain/opportunity/opportunities.db`

**系统配置检查：**
- [ ] 时区设置为 `Asia/Shanghai`
- [ ] 默认模型配置正确
- [ ] 飞书用户 ID 配置正确 (`ou_cd2f520717fd4035c6ef3db89a53b748`)

---

### 第二轮：功能性检查 (完成后)

**技能功能验证：**
- [ ] 技能能被正确触发（测试触发词）
- [ ] 技能能访问所需数据库
- [ ] 技能能写入日志文件
- [ ] 错误处理逻辑完整

**任务执行验证：**
- [ ] 任务状态正确更新 (queued → running → completed)
- [ ] 执行结果写入数据库
- [ ] 反思记录自动生成
- [ ] 飞书通知发送成功（如需要）

**监控脚本验证：**
- [ ] 脚本能正常启动（nohup + python3）
- [ ] PID 写入日志
- [ ] 轮询间隔合理（不占用过多资源）
- [ ] 异常处理逻辑完整

---

### 第三轮：用户视角检查 (交付前)

**用户体验验证：**
- [ ] 命令/操作有清晰的文档说明
- [ ] 错误消息友好且可操作
- [ ] 成功/失败有明确反馈
- [ ] 飞书通知格式清晰（使用 Emoji 和分段）

**安全检查：**
- [ ] 没有硬编码的敏感信息（API Key 等）
- [ ] 文件操作有备份机制
- [ ] 数据库操作有事务保护
- [ ] 外部调用有超时限制

**可维护性验证：**
- [ ] 代码有清晰注释
- [ ] 日志记录详细且结构化
- [ ] 配置文件有注释说明
- [ ] 有回滚/恢复方案

## 🚨 禁止行为（红线）

**绝对不允许：**
1. ❌ 未检查就声称"100% 完成"
2. ❌ 跳过任何一轮自检
3. ❌ 使用旧路径（`~/.openclaw/memory/`）
4. ❌ 修改核心配置不通知用户
5. ❌ 执行外部操作（邮件、推文）不确认
6. ❌ 忽视用户抽查发现的问题

**违规后果：**
- 立即记录到反思数据库
- 生成详细的失败分析报告
- 制定具体的改进行动
- 更新检查清单防止再犯
